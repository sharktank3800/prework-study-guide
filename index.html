HTML

  • Elements that are nested inside other elements are called child elements. If there are three elements nested inside of a body element, those three nested elements are known as child elements. The element that holds all the children—in this case, the body element—is known as the parent element.
  • An element is a type of content on a webpage.
  • The head element contains information or metadata that helps the browser to render the page correctly.
  • Metadata is data about our webpage that the browser uses to effectively serve our webpage to users.
  • The head element plays a role as a container of metadata and the meta tags contains information about the page that is used by the browser.
  • The body element represents the visible content shown to the user.
  • An HTML attribute defines additional information about an element.
  • The metadata tags are always inside the head element
  • Inside head element the title element which is a metadata tag defines the title for a webpage this element is required and is extremely important for search engine optimization (seo)
  • Inside head element the style element which is a metadata tag defines the css styles associated with a website
  • Inside head element the link element which is a metadata tag connects an external resource to the html document
  • Inside head element the meta element which is a metadat tag defines metadata such as the character set, description, keywords, author, and viewport
    • The charset attribute sets our character encoding.
    • The http-equiv attribute sets an HTTP header for content.
    • The viewport attribute sets the default view for our webpage.
  • Inside head element the script element which is a metadata tag embeds client-side scripts in the html document.
  • An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.
  • The p element represents a paragraph or block of text.
  • The ul, ol, and li elements represent unordered lists, ordered lists, and list item.
  • The img element can contain various attributes; the src attribute is the most important because it defines the location of the image.
  • The br element creates a line of empty space, or a line break, between two blocks of content.
  • The a element (which stands for anchor) creates links to the same webpage or other webpages. The href attribute points to the URL for the link.
  • The header appears at the top of the page and usually contains a navigation menu and/or introductory content.
  • The footer appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage.

CSS

  • style an element directly in the HTML file, they use inline CSS. Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only. They are useful for quick and permanent changes but are less flexible than external and internal style sheets, because each inline style must be edited individually whenever you make a design change.
  • It is also possible to embed an entire stylesheet directly in the HTML file using a style element in the head element. This internal style sheet contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code. Again, a single change to the CSS rule will apply to all tagged page elements.
  • Inline CSS and internal style sheets are totally acceptable methods under certain conditions, but the most common way to style a webpage is by using an external CSS style sheet. This is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use.
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • A margin property indicates how much space we want around the outside of an element.
  • The padding property adds space around the content inside an element. When you increase padding, you are actually making the entire element bigger.
  • it is important to know that every HTML element is essentially a box. Each box consists of margins outside the element, and the element's borders, padding, and content. This is sometimes referred to as the CSS box model.
  • We combined two selectors into a single CSS rule with a comma, allowing the declarations to affect both selectors.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.

GIT

  • version control systems can save the day! Version control systems allow us to track and manage changes to our codebase. That way, if we need to make changes or even revert back to an earlier copy, we can do so without losing other versions of our code.
  • Using version control, then, not only allows us to track our progress in our code development, but also keeps the different, older versions in safe-keeping, in case we need to revert if a mistake is made.
  • to collaborate with others, you need to use another tool, GitHub.
  • Git is version control software and GitHub is a cloud-based service that hosts our repositories which is where the project code will live.
  • Git commands always start with git and are used to save and track versions of your code. Using Git commands, we can initiate (or create) a repo, add our code to the repo, and view the different versions of our code, which are each stored in a unique commit.
  • Repository (repo) — a folder in which all files and their version histories are stored.
  • Branch — a workspace in which you can make changes that wont affect the live site.
  • Markdown (.md) — a way to write in Github that converts plain text to GitHub code. Sites such as Atom and Sublime Text are examples of free resources for developers using Markdown.
  • Commit Changes — a saved record of a change made to a file within the repo.
  • Pull Request (PR) — the way to ask for changes made to a branch to be merged into another branch that also allows for multiple users to see, discuss and review work being done.
  • Merge — after a pull request is approved, the commit will be pulled in (or merged) from one branch to another and then, deployed on the live site.
  • Issues — how work is tracked when using git. Issues allow users to report new tasks and content fixes, as well as allows users to track progress on a project board from beginning to end of a specific project.
